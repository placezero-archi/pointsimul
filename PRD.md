## 1. 프로젝트 개요

### 1.1 프로젝트명
게임포인트 시뮬레이터 (Game Point Simulator)

### 1.2 목적
게임포인트 적립 설정값에 따라 통화별 최저가 상품 결제 시 실제로 적립되는 포인트를 사전에 시뮬레이션하여, 게임 사업 부서 담당자가 포인트 정책 수립 시 데이터 기반 의사결정을 할 수 있도록 지원

### 1.3 배경
- 게임포인트 시스템은 통화별로 소수점 처리 방식이 다름
- 적립률 적용 후 시스템 저장값과 유저 표시값 간 차이가 발생
- 이로 인한 손해율을 사전에 파악하여 정책 수립 필요
- 내부 담당자들이 엑셀 등에서 수동 계산하던 프로세스 자동화

### 1.4 사용 대상
- **Primary:** 게임 사업 부서 담당자
- **Secondary:** 포인트 정책 수립 관련자

## 2. 기능 명세

### 2.1 입력 기능

#### 2.1.1 적립률 설정
- **항목명:** 적립률 (%)
- **타입:** Number (소수점 1자리)
- **범위:** 0 이상
- **기본값:** 5
- **적용 범위:** 모든 통화 공통

#### 2.1.2 통화별 설정
각 통화별로 두 가지 값을 입력:

##### 최소 상품 금액
- **정의:** 해당 통화에서 가장 낮은 가격의 상품 금액
- **타입:** Number
- **통화별 입력 단위:**
  - KRW, JPY, TWD: 정수
  - USD, EUR, THB, PHP: 소수점 둘째자리

##### 최소 사용 단위 (GP)
- **정의:** 유저가 게임포인트를 사용할 수 있는 최소 단위
- **타입:** Number (소수점 둘째자리)
- **용도:** 참고용 (현재 버전에서는 계산에 직접 영향 없음)

### 2.2 지원 통화

| 통화 | 코드 | 이름 | 심볼 | 소수점 자리수 | 기본 최소 상품 금액 |
|------|------|------|------|---------------|---------------------|
| 한화 | KRW | 한화 | ₩ | 0 | 1000 |
| 달러 | USD | 달러 | $ | 2 | 0.99 |
| 유로 | EUR | 유로 | € | 2 | 0.99 |
| 엔화 | JPY | 엔화 | ¥ | 0 | 100 |
| 대만달러 | TWD | 대만달러 | NT$ | 0 | 30 |
| 태국바트 | THB | 태국바트 | ฿ | 2 | 30 |
| 필리핀페소 | PHP | 필리핀페소 | ₱ | 2 | 50 |

### 2.3 계산 기능

#### 2.3.1 계산 트리거
- "계산하기" 버튼 클릭 시 즉시 계산

#### 2.3.2 계산 로직

**Step 1: 계산값 (Raw Value)**
```
계산값 = 최소 상품 금액 × (적립률 / 100)
```
- 아무런 처리 없이 순수 계산 결과

**Step 2: 시스템 적립값 (System Value)**
```
시스템 적립값 = floor(계산값 × 1000) / 1000
```
- 소수점 넷째자리 버림
- 소수점 셋째자리까지 저장
- 실제 서버에 저장되는 값

**Step 3: 유저 출력값 (User Display Value)**
```
유저 출력값 = floor(시스템 적립값 × 100) / 100
```
- 소수점 셋째자리 버림
- 소수점 둘째자리까지 표시
- 유저에게 실제로 보여지는 값

**Step 4: 손해율 (Loss Rate)**
```
손해율 = ((시스템 적립값 - 유저 출력값) / 시스템 적립값) × 100
```
- 시스템 저장값 대비 유저 표시값의 손실 비율
- 퍼센트(%) 단위

#### 2.3.3 계산 예시

**예시 1: 달러 $0.99, 적립률 5%**
- 계산값: 0.99 × 0.05 = 0.0495 GP
- 시스템 적립값: 0.049 GP (넷째자리 버림)
- 유저 출력값: 0.04 GP (셋째자리 버림)
- 손해율: (0.049 - 0.04) / 0.049 × 100 = 18.37%

**예시 2: 한화 ₩1000, 적립률 5%**
- 계산값: 1000 × 0.05 = 50.00 GP
- 시스템 적립값: 50.000 GP
- 유저 출력값: 50.00 GP
- 손해율: 0.00%

### 2.4 출력 기능

#### 2.4.1 결과 테이블
7개 통화에 대해 각각 다음 정보를 표시:

| 컬럼명 | 설명 | 포맷 |
|--------|------|------|
| 통화 | 통화 이름 | 한글 (예: "한화") |
| 최소 상품 금액 | 입력한 상품 금액 | 통화 심볼 + 금액 (예: "₩1000") |
| 계산값 | 적립률 적용한 원본 값 | 숫자 + " GP" |
| 시스템 적립 | 시스템에 저장되는 값 | 숫자(셋째자리) + " GP" |
| 유저 출력 | 유저에게 표시되는 값 | 숫자(둘째자리) + " GP" |
| 손해율 | 손실 비율 | 숫자 + "%" |

#### 2.4.2 숫자 포맷 규칙

**통화 금액:**
- 소수점 없는 통화 (KRW, JPY, TWD): 정수 표시
- 소수점 있는 통화 (USD, EUR, THB, PHP): 둘째자리까지 표시

**게임포인트:**
- 계산값, 유저 출력: 소수점이 0이면 정수로 표시, 아니면 둘째자리까지
- 시스템 적립: 소수점이 0이면 정수로 표시, 아니면 셋째자리까지

**손해율:**
- 항상 소수점 둘째자리까지 표시
- 0보다 크면 빨간색, 0이면 회색

#### 2.4.3 설명 섹션
테이블 하단에 각 항목 설명:
- **계산값:** 적립률 적용한 원본 값
- **시스템 적립:** 소수점 셋째자리까지 저장되는 값
- **유저 출력:** 유저에게 표시되는 값 (소수점 둘째자리)
- **손해율:** 시스템 적립 대비 유저 출력의 손실 비율

### 2.5 엑셀 복사 기능

#### 2.5.1 기능 개요
- 계산 결과를 엑셀/구글 시트에 바로 붙여넣을 수 있도록 클립보드에 복사

#### 2.5.2 버튼 위치
- 계산 결과 테이블 우측 상단
- "계산 결과" 제목과 같은 줄

#### 2.5.3 버튼 텍스트
- 기본: "계산 결과 복사"
- 복사 성공 시: "✓ 복사 완료!" (2초간 표시 후 원래대로)

#### 2.5.4 안내 문구
버튼 바로 아래:
- "엑셀에 붙여넣기하세요" (회색, 작은 글씨)

#### 2.5.5 복사 포맷
- **형식:** TSV (Tab-Separated Values)
- **구조:**
  ```
  통화\t최소 상품 금액\t계산값\t시스템 적립\t유저 출력\t손해율
  한화\t₩1000\t50 GP\t50 GP\t50 GP\t0.00%
  ...
  ```

#### 2.5.6 사용 플로우
1. "계산하기" 버튼 클릭하여 결과 생성
2. "계산 결과 복사" 버튼 클릭
3. 엑셀/구글 시트 열기
4. 원하는 셀에 Ctrl+V (또는 Cmd+V) 붙여넣기
5. 표 형태로 자동 정렬됨

## 6. 의사결정 기록

### 6.1 적립 단위 vs 사용 단위

**결정:**
- 적립 단위: 통화별 소수점 자리수와 동일
- 사용 단위: 별도 입력 (서비스 정책에 따라 다를 수 있음)

**논의 과정:**
1. 처음에는 적립 단위도 설정 가능하게 할지 고민
2. 통화 체계와 일관성 유지가 중요하다고 판단
3. 사용 단위만 별도로 설정 가능하도록 결정

### 6.2 소수점 처리 방식

**결정:**
- 계산값 → 시스템 적립값: 소수점 넷째자리 버림
- 시스템 적립값 → 유저 출력값: 소수점 셋째자리 버림

**논의 과정:**
1. 대부분의 포인트 시스템은 내림(Floor) 처리
2. 회사 입장에서 비용 예측 가능
3. 업계 표준에 따라 내림 방식 채택

### 6.3 UI 표시 방식

**결정:**
- 소수점이 0일 때 정수로 표시

**예시:**
- 50.00 GP → 50 GP
- 0.049 GP → 0.049 GP (의미 있는 소수점은 유지)

**논의 과정:**
1. 처음엔 항상 고정 자리수 표시 고려
2. 가독성 테스트 결과 0일 때 불필요하다고 판단
3. 의미 있는 소수점만 표시하기로 결정

### 6.4 엑셀 복사 기능

**결정:**
- TSV 형식 채택
- CSV가 아닌 이유: 통화 심볼(특수문자) 처리 이슈

**논의 과정:**
1. CSV vs TSV 비교
2. 엑셀에서 Tab 구분자가 더 안정적
3. TSV 채택
